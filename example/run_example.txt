Example of running XookSuut on a simulated velocity map with an oval distortion.

Inputs on square brackets are optional depending on the adopted kinematic model, or error estimation method.

USE: XookSuut name vel_map.fits [error_map.fits,SN] pixel_scale PA INC X0 Y0 [VSYS] vary_PA vary_INC vary_X0 vary_Y0 vary_VSYS ring_space [delta] Rstart,Rfinal frac_pixel inner_interp  kin_model fit_method N_it [R_bar_min,R_bar_max] [errors] [stepsize_mcmc] [use_mcmc_res=0,1] [plot_chain_mcmc=0,1] [e_ISM] [survey] [config_file] [prefix]


Description of the input parameters for the example file :

#Objects_name : example

#Velocity_map : example.fits.gz

#Error_velocity_map: -

#Pixel_scale: 0.5 arcsec

#Guess values for the constant parameters are:
#PA, INC, XC, YC, VSYS = 50, 60, 50, 50, -

#Constant parameters are allowed to vary: (1 : True, 0 : False)
#vary_PA, vary_INC, vary_XC, vary_YC, vary_VSYS = 1, 1, 1, 1, 1
 
#Spacing between rings on the disk plane is 1.5 arcsec. 

#The first ring starts in 1.5 arcsec and the last ring to explore is positioned at 100 arcsec.

#At least 1/3. of pixels are required to compute the stacked velocities.

#Velocities interior the first ring grow linearly from zero.

#kinematic_model: circular

#Least square minimization method: LM (Levenberg-Marquardt)

#N_iterations : 5

# Survey: test 



If errors are computed via MCMC and you want to use MCMC "best fit values" set use_mcmc_res = 1. 

On a terminal, run the following commands depending on the desired model and erros computation.

# For circular model without errors computation (i.e., Least Squares covariance errors)
XookSuut example example.fits.gz - 0.5 50 60 50 50 - 1 1 1 1 1 1.5 - 1.5,100 1/3. 0 circular LM 5 - - - - - - test - -

# For radial model with 5 bootstrap iterations, and noncircular flows starting at 1.5arcsec up to 20arcsec:
XookSuut example example.fits.gz - 0.5 50 60 50 50 - 1 1 1 1 1 1.5 - 1.5,100 1/3. 0 radial LM 5 1.5,20 [1,5] - - - - test - -

# For bisymmetric model using MCMC, with 100 steps, thin = 10, burin = 10 and 500 walkers, and noncircular flows starting at 1.5arcsec up to 20arcsec: 
# The minimum and maximum lenght of the bar is R_bar_min,R_bar_max = 1.5arcsec,20 arcsec
XookSuut example example.fits.gz - 0.5 50 60 50 50 - 1 1 1 1 1 1.5 - 1.5,100 1/3. 0 bisymmetric LM 5 1.5,20 [2,100,10,10,500] - - - - test - using_emcee

# For a second order harmonic decomposition model using MCMC and bootrstap errors, with 100 steps, thin =10, burin = 10 and 500 walkers, and noncircular flows starting at 1.5arcsec up to 20arcsec:
XookSuut example example.fits.gz - 0.5 50 60 50 50 - 1 1 1 1 1 1.5 - 1.5,100 1/3. 0 hrm_2 LM 5 1.5,20 [3,100,10,10,500] - - - - test - using_emcee_and_boots

# For circular model using M-H  with 10000 steps, thin =10, burin = 10
XookSuut example example.fits.gz - 0.5 50 60 50 50 - 1 1 1 1 1 1.5 - 1.5,100 1/3. 0 radial LM 5 - [4,1e4,10,10] - - - - test - using_MH






Output files are stored in the /models and /plots directories.

description of ouput files for circular model:

best line of sight velocity model: 		example.circular.2D_vlos_model.fits.gz
residual map: 					example.circular.residual.fits.gz
individual velocities with errors :		example.circular.1D_model.fits.gz
2D representation of rot. curves:		example.circular.2D_circ_model.fits.gz
chi-square map:					example.circular.chisq.fits.gz
deproject distance map (arcsec): 		example.circular.2D_R.fits.gz







The config_file allows the user to put boundaries in the constant parameters for the Least-squares analysis.
A configuration file has the follow entrance.

#XookSuut configuration file.
#
# 
# Example of configuration file
# (0) param. Description of the constant parameters. Do not touch this column !!
# (1) val. Initial values of the constant parameters.
# (2) fit. Boolen. If 1 the parameter is fitted. If 0 the parameter remains constant during the Least square analysis.
# (3) min. Minimum value for the considered parameter. If fit was set to 0, the min value is ignored. 
# (4) max. Maximum value for the considered parameter. If fit was set to 0, the max value is ignored. 
#
#
#
param	val	fit	min	max
pa	162	1	0	360
inc	48	1	10	80
x0	60.5	1	0	120
y0	65.2	1	0	120
Vsys	2000	1	0	3e10
phi_b	50	1	0	180


Running XookSuut with a configuration file

XookSuut example example.fits.gz - 0.5 50 60 50 50 - 1 1 1 1 1 1.5 - 1.5,100 1/3. 0 circular LM 5 - - - - - - test ../file.config -
